# 技能系统规范 - 任务分解

## Phase 1: 技能文件格式与解析

### Task 1.1: 定义技能数据模型
- 扩展 `internal/models/skills.go` 中的 Skill 结构体
- 添加版本、分类、标签、优先级、时间戳等字段
- 定义 Front Matter 对应的结构体

### Task 1.2: 实现 Front Matter 解析器
- 在 `internal/services/s3_skill_loader.go` 中添加 YAML Front Matter 解析
- 支持解析后验证必填字段
- 处理解析错误和格式错误

### Task 1.3: 实现技能验证器
- 验证 id 格式（小写字母数字下划线，1-64字符）
- 验证 version 格式（语义化版本）
- 验证必填字段是否存在
- 验证 keywords 非空

---

## Phase 2: 技能检索与匹配算法

### Task 2.1: 文本归一化
- 实现小写转换
- 实现中文分词（可使用简单的关键词匹配）
- 实现停用词过滤
- 实现标点符号去除

### Task 2.2: 多维度评分
- 实现关键词匹配评分
- 实现描述文本相似度评分
- 实现显示名称匹配评分
- 实现加权综合评分

### Task 2.3: 过滤与排序
- 实现阈值过滤
- 实现按评分降序排序
- 实现优先级调整
- 实现 Top-N 截断

### Task 2.4: SkillRegistry 重构
- 重构 `internal/services/skill_registry.go`
- 将现有匹配逻辑替换为新算法
- 保持向后兼容

---

## Phase 3: 技能执行上下文注入

### Task 3.1: 技能内容注入
- 在 `internal/services/skill_executor.go` 中实现上下文注入
- 将技能 Markdown 内容作为 System Message
- 保持原始系统指令
- 验证注入后的 Prompt 结构

### Task 3.2: 注入效果验证
- 验证技能描述正确影响 LLM 输出
- 验证系统指令与技能内容不冲突
- 性能测试（注入开销）

---

## Phase 4: 技能创建流水线完善

### Task 4.1: 元数据自动生成
- 在 `internal/services/skill_creator.go` 中实现
- 自动生成 id、name、display_name
- 自动提取 keywords 和 tags
- 自动设置版本和时间戳

### Task 4.2: 输出层内容审查
- 实现生成内容的注入检测
- 实现内容格式验证
- 实现敏感内容过滤（可选）

### Task 4.3: 两阶段提交完善
- 确保占位文件创建失败时的处理
- 确保 AI 生成失败时的清理
- 确保上传失败时的回滚逻辑

---

## Phase 5: 版本管理与分类

### Task 5.1: 版本号解析与比较
- 实现 SemVer 解析
- 实现版本比较函数
- 实现在加载时去重

### Task 5.2: 分类与标签体系

- 实现按分类筛选方法
- 实现标签提取和展示

## Phase 6: 缓存与性能优化

### Task 6.1: 启动加载优化
- 实现并行加载技能文件
- 实现加载进度日志
- 实现加载错误统计
